## Comandos Especializados

### Coordenação de Agentes
- `*agent pm`: Mudar para Product Manager para planejamento estratégico
- `*agent architect`: Mudar para Architect para decisões técnicas
- `*agent dev`: Mudar para Developer para implementação
- `*agent qa`: Mudar para QA para validação e testes
- `*agent analyzer`: Mudar para Analyzer para revisão crítica
- `*agent voice-assistant-specialist`: Mudar para especialista em voz
- `*agent behavioral-analysis-specialist`: Mudar para especialista em análise comportamental
- `*agent trend-prediction-specialist`: Mudar para especialista em predição

### Planejamento e Coordenação
- `*plan voice-assistant`: Criar plano coordenado para implementação de voz
- `*plan behavioral-analysis`: Planejar análise comportamental com privacy
- `*plan trend-prediction`: Estratégia para sistema de predição
- `*plan integration`: Coordenar integração de todos os componentes

### Análise e Review
- `*review current-status`: Analisar status atual e próximos passos
- `*review code-quality`: Coordenar revisão de qualidade de código
- `*review architecture`: Validar decisões arquiteturais
- `*review educational-impact`: Avaliar impacto educacional

### Implementação Coordenada
- `*implement phase-3`: Coordenar implementação completa da Fase 3
- `*implement voice-first`: Priorizar assistente de voz
- `*implement privacy-framework`: Coordenar framework de privacidade

## Estratégias de Coordenação

### Para Componentes Complexos
1. **Planning Session**: PM + Architect + Specialist definem requisitos
2. **Technical Review**: Architect + Analyzer validam arquitetura
3. **Implementation**: Dev + Specialist implementam funcionalidade
4. **Quality Gate**: QA + Analyzer validam qualidade
5. **Integration**: Full team integra e testa

### Para Decisões Críticas
1. **Multi-agent consultation**: Consultar múltiplos especialistas
2. **Risk assessment**: Analyzer identifica riscos potenciais
3. **Educational validation**: PM valida benefício educacional
4. **Technical feasibility**: Architect confirma viabilidade
5. **Implementation planning**: Dev estima esforço e timeline

### Para Quality Assurance
1. **Code review cascade**: Dev → Analyzer → QA
2. **Educational testing**: QA valida impacto educacional
3. **Performance validation**: Architect + QA verificam performance
4. **Security review**: Analyzer + QA verificam segurança
5. **User acceptance**: PM + QA validam com usuários
